# 课搭子 头像上传功能完整设计方案

> 移动端优先响应式，严格遵循项目自定义 MD3 规范  
> 创建日期：2026-06-10

---

## 一、方案概述

### 1.1 功能定位
为个人中心页面提供点击触发式头像上传功能，支持从相册选择、拍照上传和恢复默认头像三种方式，通过强制裁剪保证头像在所有 13 个使用场景下的一致性和清晰度。

### 1.2 设计原则
- **用户认知优先**：采用行业通用的交互模式，无学习成本
- **移动端优先**：所有交互和布局优先适配手机端，再扩展至桌面端
- **一致性**：严格使用项目已有的设计 Token 和 UI 组件规范
- **容错性**：完善的异常处理和错误提示，防止用户操作失败
- **安全性**：前端多重校验，弥补后端仅校验扩展名的安全缺陷

### 1.3 适用范围
- 平台：移动端（<768px）、平板（768px-1199px）、桌面端（≥1200px）
- 用户：所有已登录用户
- 入口：个人中心页面顶部的用户头像

## 二、入口设计（个人中心页面）

### 2.1 基础样式规范

| 元素 | 移动端 | 平板 | 桌面端 | 通用属性 |
|------|--------|------|--------|----------|
| 头像尺寸 | 72×72px | 64×64px | 80×80px | 圆形 (`border-radius: var(--radius-pill)`) |
| 边框 | - | - | - | `1px var(--md-outline-variant)` |
| 可点击区域 | 88×88px | 88×88px | 88×88px | 超出头像部分透明，符合 MD3 最小触摸目标 |

### 2.2 默认头像规范
- 内容：用户昵称首字母大写
- 背景色：`var(--md-primary-container)`
- 文字颜色：`var(--md-primary)`
- 文字样式：字号 24px，字重 500，居中对齐

### 2.3 交互状态设计

| 状态 | 样式表现 |
|------|----------|
| 默认 | 正常显示头像或首字母，无额外效果 |
| 悬停（桌面端专属） | 叠加 45% 透明度黑色遮罩 `rgba(0,0,0,0.45)`，中央显示 24×24px 白色相机图标 |
| 点击（全平台） | 头像缩放至 95%，弹出底部动作菜单 |
| 禁用 | 灰度显示，`cursor: not-allowed`，不响应点击 |

## 三、底部动作菜单设计

### 3.1 菜单选项

| 顺序 | 选项 | 图标 | 平台 | 说明 |
|------|------|------|------|------|
| 1 | 从相册选择 | `image` | 全平台 | 打开系统文件选择器 |
| 2 | 拍照上传 | `camera_alt` | 仅移动端 | 调用设备摄像头 |
| 3 | 恢复默认头像 | `refresh` | 全平台 | 仅当前有自定义头像时显示 |
| 4 | 取消 | 无 | 全平台 | 关闭菜单 |

### 3.2 菜单样式规范
- 圆角：顶部 `var(--radius-2xl)` (28px)，底部 0px
- 内边距：16px
- 菜单项高度：56px（标准触摸高度）
- 图标与文字间距：16px
- 分隔线：`1px var(--md-outline-variant)`
- 点击反馈：`var(--md-surface-container-lowest)` 背景涟漪

### 3.3 恢复默认头像确认流程
1. 用户点击"恢复默认头像"
2. 弹出居中确认对话框（`openModal`）
3. 标题："恢复默认头像"，内容："确定要恢复为系统默认头像吗？此操作不可撤销。"
4. 按钮：左侧"取消"（`btn-secondary`），右侧"确定"（`btn-primary`）
5. 确认后调用 `PUT /api/auth/me` 传 `{ avatar_url: "" }`

## 四、核心裁剪界面设计

### 4.1 界面布局

#### 移动端（<768px）
- 全屏显示，隐藏所有其他界面元素
- 顶部应用栏：高度 56px，黑色背景
  - 左："取消" 文字按钮
  - 中：标题 "裁剪头像"，白色，20px，字重 500
  - 右："保存" 填充按钮（主色调）
- 裁剪区域：占屏幕高度 75%，黑色背景
- 底部操作栏：高度 64px，居中旋转按钮（48×48px 圆形）

#### 平板/桌面端（≥768px）
- 居中模态对话框：600×700px
- 背景：半透明黑色遮罩
- 圆角：顶部 `var(--radius-2xl)`，底部 `var(--radius-md)`
- 内部布局与移动端一致

### 4.2 强制裁剪规则

| 规则 | 值 | 说明 |
|------|-----|------|
| 裁剪比例 | 1:1 固定 | 禁止调整 |
| 裁剪框行为 | 固定不可动 | `cropBoxMovable: false, cropBoxResizable: false` |
| 边界限制 | `viewMode: 1` | 图片永不超出裁剪框 |
| 最小裁剪尺寸 | 192×192px | 保证 24px 侧边栏 8 倍缩放清晰 |
| 初始状态 | `autoCropArea: 1` | 图片自动填充裁剪框 |
| 辅助线 | `guides: true` | 九宫格 |
| 裁剪框颜色 | `#4A90D9` | 项目主色调 |

### 4.3 裁剪交互
- 单指拖动图片 → 调整裁剪位置
- 双指捏合 → 缩放图片（最小填满裁剪框，最大 2 倍原始尺寸）
- 旋转按钮 → 顺时针 90°
- 实时预览裁剪效果

## 五、上传与状态反馈

### 5.1 上传中
- "保存"按钮禁用 + 圆形进度指示器
- 按钮文字变为 "正在上传..."
- 禁止其他按钮点击，防重复提交

### 5.2 成功
- 自动关闭裁剪界面
- 头像 300ms 淡入动画更新
- 侧边栏头像同步更新
- Toast："头像更新成功"（2.5s）

### 5.3 失败
- 保留裁剪界面
- "保存"按钮恢复可点击
- 对应 Toast 提示：
  - "头像不能超过 2MB"
  - "仅支持 jpg/jpeg/png/gif/webp 格式"
  - "图片处理失败，请重试"
  - "网络错误，请检查网络后重试"

## 六、异常情况处理

### 6.1 文件校验异常
- 文件过大（>2MB）→ Toast，不进裁剪界面
- 格式错误 → Toast，不进裁剪界面
- 图片损坏 → Toast "图片处理失败，请重试"

### 6.2 权限异常
- 权限被拒 → Toast "需要相册权限才能上传头像"
- 提供"去设置"按钮 → 跳转系统权限设置页

### 6.3 网络异常
- 网络断开 → Toast "网络错误，请检查网络后重试"
- 保留已选图片和裁剪范围，可重试

## 七、技术选型与实现要求

### 7.1 裁剪库
- **cropperjs v1.6.2**（非 v2.x）
- 理由：无依赖、轻量（28KB gzipped）、API 稳定、移动端适配好
- 方式：npm 安装 → 复制 `cropper.min.js` + `cropper.min.css` 到 `public/lib/cropperjs/`

### 7.2 前端处理
- **EXIF 方向修正**：读取 EXIF Orientation 并在 Cropper 初始化前旋转图片
- **图片预压缩**：裁剪后 canvas 输出 JPEG 质量 0.9
- **MIME 校验**：验证 `file.type` 在白名单内
- **内存管理**：关闭时 `cropper.destroy()` + `URL.revokeObjectURL()`

### 7.3 后端交互
- URL：`POST /api/auth/avatar`
- Headers：`Authorization: Bearer <token>`
- Body：`multipart/form-data`，字段名 `avatar`
- 成功：`{ "avatar_url": "/uploads/avatars/...", "message": "头像已更新" }`
- 失败：`{ "error": "错误消息" }`

### 7.4 现有组件调用方式
```js
// Toast
import { showToast } from '../components/ui.js';
showToast('消息');

// 模态对话框
import { openModal, closeModal } from '../components/ui.js';
openModal('标题', '<p>HTML 内容</p>');

// 底部动作菜单（本项目新增）
import { showBottomSheet } from '../components/ui.js';
showBottomSheet([...]);
```
