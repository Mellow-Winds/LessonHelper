# 鲸币（Coins）独立经济系统 ── TODO List

### 1. 鲸币账户基础设施（个人私密资产）

- **鲸币数量作为个人的私密资产展示**。它只在用户本人的个人空间（Profile）面板上克制地显示，其他人访问其公开主页时一律无法看到该数字。
    
- **调用现成图标展示鲸币**：
    
    - 使用 Google 库中的 `monetization_on`（外圆内带币符号的硬币图标）作为全局鲸币的视觉代币。
	- 按钮为“【图标】鲸币：xxx”
- **一个独立的鲸币详情二级子网页（`profile/coins`）**。在个人空间点击按钮时，网页通过模块化路由切入该子网页。
    
- **在详情页看到一目了然的流水账单**：
    
    - 账单按时间倒序排列，清晰记录每一笔鲸币的变动原因（例如：`2026-06-01 每日签到 +2`、`2026-06-02 下载资料 -2`）。
        
    - 账单类型调用现成图标区分：收入类（如签到、获赠）左侧配 `trending_up`（绿色上升箭头）；支出类（如下载、打赏）左侧配 `trending_down`（灰色下降箭头）。
        
- **二级网页标配返回机制**。详情页左上角有带 `arrow_back` 图标的顶栏面包屑按钮，允许用户随时一键无缝退回全局个人空间。
    

### 2. 连续学习与漏签挽回的动能转换

- **签到机制与鲸币产生联动**。用户每天登录并点击签到（签到按钮可使用现成图标 `task_alt` 或 `workspace_premium`），直接获得基础鲸币（如 +2）。
    
- **漏签缓冲倒计时与变现惩罚挂钩**：
    
    - **短期保护**：连续签到 7 天以上漏签，进入“3天生命倒计时”。3天内及时签到，复活连续天数。
        
    - **长期保护**：连续签到 45 天以上漏签，进入“7天生命倒计时”。7天内及时签到，全面复活天数。
        
    - **老友回归大礼包**：如果连保护期也过期了，天数彻底归零。当该同学很久之后重新回来点击签到时，不恢复天数，但触发回归礼包，直接一次性赠送10鲸币，作为他重整旗鼓的启动资金。
        

### 3. 鲸币生态圈的“开源”与“节流”（平衡性闭环）

为了防止水贴刷爆系统，在后端锁死“每日获取上限（Hard Cap）”。

- **赚取鲸币（开源）**：
    
    - 发布一门课程的自习邀约、广场帖子或上传课件资料，可获得较多鲸币（如 +2）。
        
    - **每日防刷上限**：单个账号每天通过发帖/传资料获取的鲸币，在后端死锁上限（例如每天最多通过活跃行为赚取 5 鲸币），超过后行为依旧允许，但不产生任何新货币，绝杀刷分机器人。
        
- **消耗鲸币（节流与宏观调控）**：
    
    - **有偿下载优质资料**：实行有偿下载，扣除 2 鲸币。
        
    - **系统抽成销毁（通缩机制）**：下载扣除的 2 鲸币中，1 币由系统直接分配给该资料的上传者（对其提供长期维护的物质激励），另外 1 币由系统直接回收并就地销毁。通过这种自然通缩，确保全站的鲸币总额永远处于动态平衡，绝不发生恶性通货膨胀。
        

### 4. 社交赠予与打赏机制

- **搭子之间可以相互转赠与发帖打赏**。觉得同学的离散数学笔记写得好、或者交友广场的组队宣言合适，可以在帖子下方一键点击“打赏”按钮（使用现成图标 `thumb_up` 或 `redeem`）。自己钱包扣除 X 鲸币，对方钱包无缝到账 X 鲸币（此行为属于存量流通，系统不增发新币）。
    
- **防薅羊毛防火墙（反作弊门槛）**：为了防止攻击者批量注册空白小号互相转账套现，系统增加硬性前置门槛——只有账号注册满 7 天，且通过资料填写、“资料完整度”达到 80% 以上的用户，才能解锁“转赠”和“打赏/接受打赏”功能。
