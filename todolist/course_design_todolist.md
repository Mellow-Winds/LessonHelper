# 课程板块双轨制重构 ── 终极版 TODO List

### #1. 侧边栏设计（Sidebar Layout）

- **分类大类归属**：将侧边栏的“课程列表”完全剥离，在【功能】栏目下并列部署两个新菜单项，平齐原有的 `search` 和 `explore`。
    
- **【我的课程】展现**：路由为 `*/mycourse`，点击后右侧主舞台渲染当前学期课表。
    
- **【课程广场】展现**：路由为 `*/allcourse`，点击后右侧主舞台渲染全校学科档案馆。
    

### #2. 我的课程（My Courses - 班级视角）

#### 2.1 页面总览

- 与现有课程页面保持大体一致，按教务系统的标准学期标头进行物理聚合。
    

#### 2.2 课程空间（小班空间 - 详情页）

- **顶部面包屑与标题**：显示当前的班级名，例如：`我的课程 / 离散数学03班`。
    
- **右上角双控制流**：
    
    - **【跳转大课】按钮**：使用线条图标 `shortcut` 或 `arrow_forward`，点击立刻跨越时空跳转至对应的大课空间。
        
    - **【+ 发布】动作按钮**：调用线条图标 `add`，点击后直接跳转至发布系统。
        
- **药丸级面板切换器（Pill Tabs）**：在标题下方水平排列三个极其干净的 MD3 药丸形状按钮：`[ 论坛 ]`、`[ 资料 ]`、`[ 成员 ]`。点击其中一个按钮高亮，下方主容器流式渲染对应内容：
    
    - **【论坛】面板呈现**：展示当前学期该班级内同学发的讨论帖、求助帖或水贴。支持筛选，且筛选选项与发布的官方固定分类完全等价锁死。
        
    - **【资料】面板呈现**：平铺展示本学期同学们上传的本地课件、作业 PDF。每一项右侧有下载和收藏按钮。
        
    - **【成员】面板呈现**：小班核心功能。平铺渲染当前班级内所有同学的扁平化社交名片，包含头像、姓名、专业。点击同学名片可直接穿梭查看其【公开主页视角】，极易在该班级内定向组队或寻找自习搭子。
        

### #3. 课程广场（All Courses - 跨届大课视角）

#### 3.1 页面总览

- 和现在的“课程页面”保持大体一致。
    
- **大类名称生成与映射规则**：建立从小班到大课广场的全局截断映射机制。系统在识别和匹配大类课程时，**统一全量清除文本末尾的班级号标识（如物理去除“07班”、“03班”、“10班”或“25”）**。但是，**课程标题中自带的教学层次限定词必须无条件强行保留**。例如：
    
    - `学术性阅读：语言与思维07班` 自动剥离映射为大类：`学术性阅读：语言与思维`
        
    - `需求与商业模式创新03班` 自动剥离映射为大类：`需求与商业模式创新`
        
    - `微积分II(第一层次)10班` 自动剥离映射为大类：`微积分II(第一层次)`（括号内的层次文字必须完好保留）
        
    - `软件工程与计算 I03班` 自动剥离映射为大类：`软件工程与计算 I`
        
- **大类基础索引源（内置预留）**：系统底层预留对全校核心通识与专业课程的权威映射索引表 `allcourses.xlsx`。未来开发完成后，将原封不动全局导入该表格作为大课的底层标签、名称规范和分类依据，所有的全局检索与大课卡片生成均以此表格数据为准进行匹配。
    
- **首要全局检索框**：页面顶部中央常驻一个高显眼度的长条状搜索栏，左侧带 `search` 放大镜图标。支持基于学科大类一览表导出的标准名称（如输入“微积分”）实时模糊过滤全校大课。
    
- **大课去学期化卡片流**：下方不分学期、不分老师、不分班级，纯粹以学科大类卡片进行大扁平化排布。
    
- **大课卡片视觉**：每个卡片极为纯净，只包含剥离班级号、保留层次后的纯净大课名称。
    

#### 3.2 课程空间（大班空间 - 详情页）

- **顶部大标题**：仅显示学科大项名称，如《离散数学》或《微积分II(第一层次)》。
    
- **右上角单动作流（死锁验证）**：
    
    - 依然部署一个核心动作按钮：`【+ 发布】`。
        
    - **“无小课不发布”拦截机制**：点击时，前端首先校验当前用户的资产账本中，是否修读了隶属于该大课的任意一个小班（例如：检查自己是否加入了微积分II的某个特定教学班）。若未加入，按钮点击置灰熔断，并弹出 Toast 警告：“课程广场为只读档案馆。你未修读本门课程的任何班级，暂无发布权限。”
        
- **物理去耦主舞台（彻底隐去成员与数据两极分化）**：页面上绝对不加载、不渲染任何成员列表或通讯录，完全切断人际社交过载，腾出 100% 的视觉空间实行双栏数据流绝对隔离渲染：
    
    - **【历届资料文件】流呈现**：**只捞取分类为 `[资料分享]` 且带有真实附件的数据**。按时间或热度流式排布历届学长学姐上传的所有通关秘籍、期末真题、学霸笔记。每个资料卡片上明码标价“消耗 X 鲸币”，并带有下载频次锁风控。
        
    - **【大课论坛】流呈现**：**只捞取分类为 `[讨论]`、`[水贴]`、`[求助]` 的纯文本/图文动态**。展示全校所有届别跨时空同步过来的讨论、求助与开学打卡水贴。卡片边角会自动附加带有时空出处的微型灰色标签（例如：`来自 2025-2026第2学期·03班`），允许用户一键勾选过滤。
        

### #4. 发布系统（Shared Publishing Control）

- **触发入口**：无论是小班详情页（`my_courses.js`）还是大课广场详情页（`plaza.js`，通过在修验证后），页面右上角均标配统一的 `【+ 发布】` 按钮。
    
- **二级网页路由跳转**：点击后，路由统一穿梭切入 **`*/course/posts`** 独立发布子页面。
    
- **统一表单界面布局**：提供一个高内聚的整块表单舞台。页面从上到下依次平铺以下核心组件：
    
    - **官方固定分类下拉框**：锁死且仅允许用户从 `[讨论]`、`[资料分享]`、`[水贴]`、`[求助]` 中四选一，禁止自主打字输入标签。
        
    - **通用标题栏与正文框**：提供原生字体继承的输入域（少于 15 字限制提交）。
        
    - **通用附件上传槽**：常驻在正文下方的极简拖拽区域。支持一键点击或拖拽本地文件作为帖子附件，由后端中间件执行 **20MB** 的单文件物理大小和后缀名强校验。
        
- **大课同步联动状态机与资产风控**：
    
    - **强控制拦截**：当用户在下拉框中主动选择了 `[资料分享]`，或者在附件上传槽中真实添加了任意本地文件时，表单底部的“是否同步发送到大课广场”开关**强制高亮保持开启，且物理置灰锁定**（不允许用户关闭），确保全站所有附带资料的帖子无条件沉淀至大课广场档案馆。
        
    - **软同步与出处追溯**：在不带附件的纯文本 `[讨论]`、`[求助]` 或 `[水贴]` 状态下，底部的同步开关恢复自由勾选。若用户保持勾选并提交，由于发布入口或用户身份绑定了其所在的小班，数据在入库时会自动封装当前的学期与班级主键（包含提取出的大课标识和保留的课程层次），在大课广场端渲染时自动外挂带有出处的微型灰色标签。
