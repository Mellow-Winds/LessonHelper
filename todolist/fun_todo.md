# 趣味模块设计
#### 一、 “百宝箱”模块（侧边栏 / 附栏）深度规划

“百宝箱”的定位是工具集合与多巴胺自动机。除了

-  “番茄时钟”：25min的计时器。

-   今日运气值：基于用户ID和算法生成一个40-100的数字

- 替我抉择：有三种：抛硬币（正面49%，反面49%，硬币立起来2%）、掷骰子（1-6随机数），随机数（自定义范围）。
    
- **薛定谔的待办（盲盒任务）：** 每日仅可抽取一次的微型挑战。例如：“今天喝满三杯水”、“给通讯录第十个人发个表情包”、“站起来伸展一分钟”。

**技术与 UI 落地要求：**
- 采用卡片式设计，圆角遮罩处理。
- 保持极高加载速度，工具逻辑尽量完全在纯前端，用户本地计算，不增加后端 API 负担。
#### 二、 “回声洞”模块（左侧导航栏底部）深度规划

“回声洞”其核心价值在于展现产品的“灵魂与幽默感”，打破冰冷的工具感。同时用户可以留言，便于后期进行盈利。
**表现层（UI/UX）设计：**
- 一个圆角卡片遮罩，从内容库随机抽出一句话展示。
- 点击后，清除现有展示，并从内容库抽出一条新的展示，并加上类似于ppt的文字打字机动画，展现一个个乱码变成字的动画。

**内容池（内容库）构建策略：**
1. 开发者自建
2. 用户参与

# UGC设计

UGC（用户生成内容）是社区的血液。对于“发现页”的卡片式发布，不能仅仅提供一个巨大的空白文本框（这会导致信息信噪比极低），而应该采用“约束即创造（Constraint breeds creativity）”的模块化设计。
