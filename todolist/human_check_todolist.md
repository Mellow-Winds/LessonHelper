# 人机验证（Cloudflare Turnstile）全链路风控 ── TODO List

### 1. 账号入口安全防御（注册阶段）

- **我想要在用户注册界面强制嵌入人机验证卡片**。
    
- **双重验证防火墙**：用户在填写邮箱、密码并点击“发送验证码/注册”之前，必须物理通过 Turnstile 的行为检测。
    
- **后端死锁逻辑**：前端在通过验证后，随表单隐式发送验证令牌（Token）。后端必须先向 Cloudflare 远程节点核验该令牌是否合法、是否由机器人伪造，核验通过才允许触发校园邮箱验证码的发送，从源头上绝杀恶意批量注册、爆破服务器路由的脚本大军。
    

### 2. 探索板块内容发行风控（每次发帖/发布邀约）

- **我想要每一次点击“发布”时都进行人机验证**。
    
- **有感/无感流式切换**：在 `explore/posts` 网页的表单最下方、点击“确认发布”按钮的正上方，部署高定线条质感的验证卡片。
    
- **高危行为零容忍**：由于发帖和发布自习邀约直接挂钩“鲸币”发行，必须做到“一帖一验”。机器人或脚本在没有真实模拟 Turnstile 令牌核验的情况下，无论正文如何精妙，后端接口一律就地物理熔断并拒绝入库，确保社区内容 100% 由真人产出。
    

### 3. 全局高频交互风控（频繁搜索）

- **我想要对高频、恶意的搜索行为进行 Turnstile 强行拦截**。
    
- **动态触发机制（智能防御）**：正常同学正常查资料、打字时，系统仅保持低成本的前端防抖（Debounce）处理，不打扰用户体验。
    
- **阈值熔断阻断器**：如果在短时间（如 5 秒内）检测到同一个账号或同一个 IP 地址疯狂触发了超过 10 次的全局检索请求（疑似恶意爬虫或爆破接口），搜索功能立刻锁定，并在搜索框下方流式弹窗召唤出人机验证。用户必须手动滑过验证，证明自己不是爬虫，才能解锁后续的搜索权限。
    

### 4. 课程私域空间核心交互风控（论坛发起、上传资料、多次回复）

- **我想要在课程空间发起论坛讨论时，每一次都触发人机验证**。
    
- **我想要在上传复习课件、学霸笔记等静态资料时，每一次都触发人机验证**。防止恶意机器人上传带有木马病毒、违规广告的空附件榨干服务器硬盘。
    
- **高频回复拦截流（多次回复）**：在同一个帖子内，如果用户连续、快速地发表评论（如 1 分钟内连续跟帖回复超过 3 次），系统立刻判定该行为具有“灌水套现鲸币”的嫌疑。后续的每一次回复框上方，都将强制塞入人机验证，直到其行为频率回归正常人范围。
    

这份风控研判大纲把 Turnstile 部署在全站最核心的四个“增、删、改、查”高危风口上，配合后端验证，能确保未来上线后，整个平台的“鲸币生态”和服务器带宽处于绝对安全的闭环之中。
