# 基础互动板块：论坛设计规范 (MVP 极简版)

本规范聚焦于构建一个高频流畅、极致扁平的轻量化社区。在 MVP 阶段彻底剔除过度设计，有且仅保留最核心的“文字+图文流”与“无状态纯数字点赞”互动，通过基础流控确保社区防刷安全。

## #1. 视觉布局与卡片架构

全面改用**扁平流式分割线架构**，极致压缩垂直视觉空间，提升滚动浏览时的吞吐量。

- **侧向并行对齐（Avatar-Content Parallelism）**：
    
    - **左侧**：固定放置圆形头像（直径 `32px`），下方不留白，直接作为垂直定位线。
        
    - **右侧**：展现用户昵称、发布时间戳、正文文本。如果有图片，无条件呈现在文本的正下方。
        
- **右侧边缘点赞区（Pure Local Likes）**：
    
    - 每一层帖子或回复的最右侧边缘，垂直固定一个 `favorite_border`（心形图标）及下方的**纯数字计数器**。
        
    - **MVP 解耦逻辑**：点赞只做**纯数字统计的累加/累减（+1 / -1）**。数据物理保存在帖子表字段中，**不产生任何全局联动，绝对不触发通知中心（不向被点赞人发送“xxx点赞了你的帖子”消息）**，保持底层逻辑的绝对清爽。
        

## #2. 文本与图片回复流规范

- **多媒体白名单**：输入流在代码层执行强校验，**有且仅有【纯文本】与【本地图片附件（至多9个）】**。物理阻断 `.pdf`、`.zip` 等任何文档文件注入。
    
- **流式纵向排版**：图片绝不作复杂的浮动或左右包裹，**统一强行渲染在当前行内文本的垂直正下方**。如有多图（2-9张）自动触发紧凑型方正网格栅格化排布，单张图片默认设置最大高度（如 `120px`）并带有微型圆角，超出则等比缩合，点击可唤出原生高清大图。
    
- **字数限制全量释放**：**前后端完全不设任何字数下限与上限拦截**。允许用户发送单字、单个表情符号或长篇吐槽，输入框不作阻断，将防刷风控完全交由底层频次流控处理。
    

## #3. 动态折叠式楼中楼级联结构

所有的多级对线互动全部原地收拢，默认实行高比例视觉截断，拒绝产生任何多余的页面跳转。

- **首屏截断隐藏机制**：
    
    - 一级回复下方，默认**最多渲染 1~2 条**最新的二级回复作为预览。
        
    - 其余更深层次的对线内容默认隐藏，下方常驻一个触发器：`── 查看更多 X 条回复 🔽`。
        
- **行内原地流式展开**：点击后不跳页、不弹窗，直接在当前页面向下平滑推开、原地流式渲染全量回复树。
    
- **二级水平扁平化对线**：二级回复内部继续点击回复时，结构不再向右作级联缩进。所有子孙级回复在第二层水平完美对齐，通过文本首部高亮蓝链接强制锁定身份：`用户A 回复 用户B：正文内容`。点击蓝色名字单页穿梭至该同学的【公开主页视角】。
    

## #4. 基础流控与防刷机制

|**流控维度**|**阈值机制**|**前端表现（Client）**|**后端风控（Server）**|
|---|---|---|---|
|**高频冷却（Debounce）**|**30 秒 / 次**|点击发布后按钮进入 30 秒置灰倒计时，文本变为 `重新发送(30s)`。|校验 User ID 上次写入时间，30 秒内二次请求直接抛回 `429` 熔断。|
|**单日总量熔断（Hard Cap）**|**100 个 / 天**|计数器达 100 次后，发布入口物理置灰，提示：“今日发布额度已达上限”。|联表查验当前用户今日产生的总互动量，超出 100 物理熔断写入权限。|

## #5. 回复与发布机制 (Publishing & Replying Mechanics)

系统通过“行内无缝唤起”与“非遮挡式下沉面板”设计，确保用户在不丢失当前阅读上下文的前提下，实现高专注度的输入体验。

- **主贴发布控制流 (独立路由 `*/course/posts`)**：
    
    - 界面采用极简大白舞台。顶部部署官方分类选择器（Select），中部留给继承系统原生的无边框文本框。
        
    - 底部工具栏左侧为图片挂载按钮，实时显示上传计数（如 `3/9`）。多图上传在框内提供九宫格微型预览，每张缩略图右上角外挂一键物理移除的 `×` 按钮。
        
- **行内交互式回复流 (Inline Sub-Editor)**：
    
    - 在课程空间详情页中，点击任意帖子或楼层下方的“回复”按钮，**不进行页面跳转，不弹出全屏阻塞式弹窗**。
        
    - 系统直接在被点击卡片的垂直正下方，原地平滑推开并向下滑出一个高度内聚的「轻量化行内输入条」。输入条包含极简文本输入域、相册选择图标（至多 9 张）与发布按钮。用户可在参照上方对线上下文的同时进行丝滑回帖，最大化信息对齐效率。

### 6.4 核心组件排版细则对照表

|**界面组件**|**物理视觉呈现规范**|**MVP 数据与交互绑定逻辑**|
|---|---|---|
|**圆形头像 (`Avatar`)**|直径 `32px` 的纯正圆形裁剪，左侧垂直对齐。|点击直接单页路由跳转至对应的 `公开主页`。|
|**多图附件区**|紧贴在文字**垂直正下方**，2-9张自动触发等长正方形 Grid 矩阵。|物理上限 9 张；点击触发原生高清无损大图灯箱放大。|
|**右侧点赞区 (`❤️`)**|强行死锁在卡片**最右侧边缘**，由心形图标和纯数字上下组成。|纯本地数字变切计数（+1 / -1），绝对不产生通知回执。|
|**回复触发按钮 (`[chat_bubble] 回复`)**|紧跟在内容/图片下方的轻量文本图标，常驻在每一层互动卡片内。|**唤起开关**：点击后原地平滑下推 DOM，展示行内输入域。|
|**图片上传槽 (`[camera]` / `[+]`)**|嵌入在输入框右侧或下方的线条图标。|点击唤醒本地相册/相机，强校验 9 张上限与 20MB 体积。|
|**扁平化对线标识 (`↩️`)**|二级流内不缩进，首部渲染：`用户A ↩️ 用户B：`。|两个用户名均渲染为高亮蓝链接，点击均可穿梭至主页。|
|**动态折叠触发器**|一级卡片底部居中展现：`── 查看更多 X 条回复 🔽`。|行内原地平滑推开/收拢 DOM 树，绝不产生跳页。|
