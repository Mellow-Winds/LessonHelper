### 1.2 项目背景与设计初衷

结合大学生真实学习场景，本项目主要解决三大痛点：寻找同课程学习伙伴困难、课程学习资料分散杂乱、线下自习邀约与组队学习效率低下。

平台以课程为核心载体，打破传统行政班级的限制，实现跨班级资源共享与交流互动。同时配套社交互动、消息通知、实用工具、隐私管理等功能，全面满足学生课内交流、资料流转、结伴学习等日常需求。

### 1.3 核心功能详细说明

#### （1）用户系统

平台采用邮箱验证码注册、密码登录的方式，基于 JWT 完成身份认证，身份令牌有效期为7天。支持头像上传，并提供图片裁剪、三级压缩处理能力。

个人资料可填写昵称、性别、专业、年级、社交账号、MBTI、肖像文字描述等信息，页面展示资料完整度进度条。

系统设有每日签到、连续打卡、双重宽限期保护机制；内置关注与粉丝体系，可查看关注动态。

支持细粒度隐私权限设置、访客视角预览、个人数据 JSON 导出以及问题反馈功能。

#### （2）课程系统

支持导入南京大学教务系统导出的 Excel 课表文件，系统自动解析文件并合并归并同名称大课。

提供学期筛选、选课、退课、课程跨学期迁移等课程管理功能。

每门大课独立划分论坛、资料、交友三大功能分区；同时设置课程广场，聚合展示全校课程，实现跨班级内容共享。

内容发布页面搭载富文本编辑器，支持多文件附件上传与内容分类；未选课用户仅拥有课程内容只读权限。

#### （3）发现与社区模块

展示全局帖子信息流及帖子详情页面，配备完整的论坛式评论体系：支持内联编辑、图片回复、楼中楼递归评论、用户@提及功能。

评论发布设置30秒冷却限制，冷却状态通过本地存储持久保存，页面刷新后状态不会重置。

页面集成活动卡片、审批、投票、倒计时等通用组件，适配各类社区互动场景。

#### （4）学习资料模块

支持上传 PDF、PPT、Word、图片四种格式文件，单文件最大限制为20MB。

提供分类浏览、章节筛选、五星评分、按评分排序等管理功能；文件下载遵循 RFC 5987 规范，兼容中文文件名，解决下载乱码问题。

#### （5）自习邀约模块

用户可自主发布、加入、取消自习邀约，支持设置参与人数上限。

系统根据实际运行状态，自动标注招募中、已满、已关闭三种状态，便于用户直观管理。

#### （6）交友广场

平台设置七类学习相关需求帖子，所有发布的帖子7天后自动过期。

交互逻辑为：感兴趣→双向确认→展示联系方式，保障线上社交安全性，帖子下方支持评论交流。

#### （7）消息通知模块

页面顶部设置消息铃铛图标与未读消息角标，后台每30秒轮询更新消息数据。

点击通知可直接跳转至对应课程、帖子或用户页面，评论区可自动定位并高亮对应内容。

支持消息一键全部标为已读、批量删除、批量管理操作。

#### （8）全局搜索

支持分标签检索课程、资料、帖子三类内容，搜索关键词高亮展示，系统自动留存搜索历史记录。

额外搭载 AI 智能搜索模式，提升内容检索效率。

#### （9）收藏系统

支持对课程、帖子进行收藏操作，单独设立「我的收藏」页面，统一管理所有收藏内容。

#### （10）百宝箱工具集

集成多款轻量化实用工具：番茄钟、每日运势、决策助手、盲盒；同时包含回声洞功能，展示随机语录并开放用户投稿入口。

## 二、作品目标

1. 针对性解决大学生找课友难、学习资料分散、自习邀约效率低三大核心问题。
2. 通过课表导入、大课自动归并功能，打破行政班级壁垒，实现跨班级课程内容、学习资料、交流互动全面共享。
3. 搭建完整的学习互助体系，实现资料流转、论坛讨论、结伴学习等基础学习社交能力。
4. 集成轻量化效率工具，丰富平台使用场景，提升学习体验。
5. 坚持隐私优先的设计原则，通过多维度权限控制，全方位保障用户个人信息安全。

## 七、部署指南

### ⚠️ 安全提示

仓库已完成密钥清理，所有历史包含密钥的提交已重写。**如果你之前 clone 过本项目，需要删除本地仓库重新 clone**，否则 `git push` / `git pull` 将无法正常使用。

如果你有未提交的本地修改，按以下步骤迁移：

1. 把改过的文件复制到仓库外面的临时文件夹
2. 删掉旧的 `LessonHelper` 文件夹
3. 重新 clone：`git clone git@github.com:Mellow-Winds/LessonHelper.git`
4. 参照下方部署指南重新搭建 `.env`
5. 把之前备份的修改手动搬回去

详细的开发环境搭建步骤请参阅仓库内的 [README_TEST.md](../README_TEST.md)。

### 7.1 环境要求

1. Node.js 版本 ≥ 18.x
2. npm 版本 ≥ 9.x
3. 可选配置：Resend API Key（用于发送邮箱验证码）

### 7.2 部署步骤

#### 1. 克隆项目仓库

```bash
git clone git@github.com:Mellow-Winds/LessonHelper.git
cd LessonHelper
```

#### 2. 安装项目依赖

```bash
npm install
```

#### 3. 配置环境变量

在项目根目录新建 `.env` 文件，写入以下内容：

```env
# JWT 密钥 — 执行以下命令生成，每人独立生成，不要共享
# node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
JWT_SECRET=把这里替换成你生成的64位随机密钥

# Resend API Key（测试环境可留空，不影响核心功能）
RESEND_API_KEY=

# Cloudflare Turnstile 人机验证 — 测试密钥，本地开发直接用
TURNSTILE_SITE_KEY=1x00000000000000000000AA
TURNSTILE_SECRET_KEY=1x0000000000000000000000000000000AA
```

> **Cloudflare Turnstile 测试 Key 说明：**
>
> | Key 类型 | 值 | 说明 |
> |----------|-----|------|
> | Site Key（前端） | `1x00000000000000000000AA` | always pass，visible 模式 |
> | Secret Key（后端） | `1x0000000000000000000000000000000AA` | always pass，跳过云端验证 |
>
> 其他可用的 Turnstile 测试 Key：
> - `2x00000000000000000000AB` — Site Key（always fail，visible）
> - `1x00000000000000000000BB` — Site Key（always pass，invisible）
>
> 生产环境请前往 [Cloudflare Turnstile 控制台](https://developers.cloudflare.com/turnstile/) 申请正式密钥，替换 `.env` 中的测试 Key。

#### 4. 启动服务

```bash
# 开发模式运行
npm run dev

# 生产模式运行
NODE_ENV=production npm start
```

服务默认访问地址：`http://localhost:3000`

生产环境地址：<https://kedazi.top>

#### 5. 服务验证

执行以下命令校验服务运行状态：

```bash
curl http://localhost:3000/
```

页面返回 HTML 内容，即代表服务启动成功。

## 八、基础使用流程

### 1. 注册与登录

访问服务地址，点击登录/注册入口，填写邮箱并获取验证码，设置密码完成注册；登录后完善个人资料。

### 2. 导入课表

进入「我的课程」页面，选择导入课表功能，上传南京大学教务系统导出的 XLSX 文件，系统自动解析并归类课程。

### 3. 课程空间使用

进入对应课程页面，可在论坛板块发帖、评论；在资料板块上传、下载文件；在交友板块发布找课友相关帖子。

### 4. 自习与交友

进入「发现」页面，使用自习邀约、交友广场功能，发布或参与活动、需求帖子。

### 5. 消息管理

点击页面顶部铃铛图标查看消息，支持批量标为已读、批量删除，点击通知可直接跳转至对应内容位置。

### 6. 工具使用

进入「百宝箱」页面，使用番茄钟、运势、决策助手、盲盒、回声洞等功能。
