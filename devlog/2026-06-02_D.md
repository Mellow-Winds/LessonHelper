# 2026-06-02 课程系统扁平化重构

## 完成事项

### 课程系统彻底扁平化大修

- **消灭"小课空间"**：全站不再有 `mycourse-detail` 和 `plaza-course` 两个独立详情页，统一为唯一的 `course-detail` 页面。
- **新建 `detail.js`**：全站唯一的统一大课空间详情页，通过 courseId 动态评估用户选修状态，实行动态权限与面板渲染：
  - 已选修 → 三段药丸（论坛/资料/交友）+ 发布按钮激活 + 成员侧栏
  - 未选修 → 两段药丸（论坛/资料）+ 发布按钮置灰 + 交友物理隐藏 + 只读模式
- **新建 `all_courses.js`**：从原 `plaza.js` 迁移课程广场搜索功能，聚合逻辑不变，导航改为统一的 `course-detail`。
- **精简 `my_courses.js`**：仅保留选修课表列表、选课模态框、导入课程表、退出课程。详情页逻辑全部迁出。
- **重写 `publish.js`**：移除对 `window._myCourseSpace` 的依赖，返回按钮统一导航到 `course-detail`。
- **更新路由表**：`router.js` 中 `mycourse-detail` 和 `plaza-course` 路由替换为 `course-detail`。
- **更新全局函数注册**：`main.js` 的 import 和 window 注册全部配平。
- **简化搜索跳转**：`auth.js` 的 `navigateToCourseResult` 不再区分 mycourse/plaza，统一导航到 `course-detail`。
- **更新所有引用**：`following_feed.js`、`profile.js`、`notification_routes.mjs`、`my_posts.js` 中的旧页面名全部替换。
- **删除 `plaza.js`**：功能已完全迁移到 `all_courses.js` 和 `detail.js`。

### 大课名称清洗规则

- 系统统一清除课程名末尾的班级号数字与"班"字（如"07班"、"25"）
- 强行保留括号内教学层次限定词（如"(第一层次)"、"(英)"）
- 清洗函数 `cleanBigCourseName` 从 `all_courses.js` 导出，供详情页使用

### 测试更新

- 更新 6 个前端测试文件以匹配新的页面名和文件结构
- `npm test` 全部 37 个测试通过
