# 2026-05-31 开发日志

## 完成事项

### 1. 课程导入对接课程空间论坛
- `routes/schedule.js`：导入课程写入 `courses` 表，自动获得课程空间（帖子/评论/成员）
- 课程描述拼接为 `课程号 · 时间 · 地点`

### 2. 移除创建课程逻辑
- 删除前端 `openCreateCourseModal()` / `handleCreateCourse()`
- 课程创建统一通过 xlsx 表格导入

### 3. 用户独立课程列表
- `GET /api/courses` 需登录，只返回当前用户课程
- 未登录跳转登录页

### 4. 导入前使用须知
- 点击"导入课程表" → 先弹出 `pre-notes.md` 使用须知
- "不同意"关闭 / "我已同意并知晓"进入导入界面

### 5. Markdown 渲染：迁移到 markdown-it
- 替换 `marked.js` 为 `markdown-it` 包（本地化，不依赖 CDN）
- `npm install markdown-it` → 复制 `markdown-it.min.js` 到 `public/js/`
- `renderMarkdown()` 使用 `markdownit().render(text)`
- `**加粗**`、`*斜体*` 等语法现可正确渲染

### 6. 导入风控：学期阶段制（替换原 100 天冷却）
- **第一学期**（8.15 - 1.1）：可导入一次
- **寒假**（1.2 - 2.14）：禁止导入
- **第二学期**（2.15 - 6.14）：可导入一次
- **暑期**（6.15 - 8.14）：可导入一次
- 通过 `user_courses.semester_key` 字段记录导入学期
- 每学期每用户仅允许导入一次

### 7. 课程搜索：选择已有课程
- 课程列表页新增"选择已有课程"按钮
- 搜索弹窗支持：课程号（输入）、课程名称（输入）、上课时间（周一~周日选择框）、教师（输入）
- 后端 `GET /api/schedule/available` 支持多条件筛选
- 搜索结果显示"已加入"状态或"加入"按钮
- 加入时检查 50 门课程上限

### 8. 退出课程 + 课程总数限制
- 每个课程卡片显示"退出课程"按钮
- 后端 `DELETE /api/courses/:id/leave` 删除 `user_courses` 记录
- 单用户课程总数上限 50 门（导入和加入均检查）
- `user_courses.semester_key` 字段迁移（`server.js`）

## 风控数据流

```
导入课程表:
  学期阶段检查 → 已导入检查 → 课程总数检查 → 解析xlsx → 单次≤50门 → 写入DB

选择已有课程:
  搜索条件 → GET /api/schedule/available → 结果列表 → 加入 → 课程总数≤50检查

退出课程:
  确认 → DELETE /api/courses/:id/leave → 删除user_courses记录

学期筛选:
  GET /api/courses/semesters → 填充下拉框
  切换学期 → GET /api/courses?semester=xxx → 刷新列表
```
