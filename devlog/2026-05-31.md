# 2026-05-31 开发日志

## 完成事项

### 1. 课程导入对接课程空间论坛
- 修改 `routes/schedule.js`：导入接口现在将解析出的课程**写入数据库** `courses` 表
- 每门导入的课程自动获得独立的课程空间（帖子、评论、成员管理）
- 课程描述字段拼接为 `课程号 · 时间 · 地点` 格式
- 导入后自动刷新课程列表，课程卡片可点击进入论坛空间

### 2. 移除创建课程逻辑
- 删除前端 `openCreateCourseModal()` 和 `handleCreateCourse()` 函数
- 课程列表页移除"创建课程"按钮，仅保留"导入课程表"
- 空状态提示改为"点击导入课程表从 xlsx 文件导入"
- 课程创建统一通过 xlsx 表格导入实现

### 3. 导入防暴力攻击限制
- **单次上限**：一次导入最多 50 门课程，超出返回 400 错误
- **频率限制**：100 天内最多导入一次，超出返回 429 错误
- 限制在服务端 `routes/schedule.js` 中实现，内存级状态

### 4. 服务端改造
- `routes/schedule.js` 构造函数改为接收 `db` 参数
- `server.js` 中 `require('./routes/schedule')(db)` 传入数据库实例
- 导入事务使用 `BEGIN TRANSACTION / COMMIT / ROLLBACK` 保证原子性

### 5. 用户独立课程列表
- `GET /api/courses` 改为需登录，只返回当前用户拥有或加入的课程
- 未登录用户访问课程列表自动跳转登录页
- 导入课程时，课程绑定当前用户（`owner_id` + `user_courses` 记录）
- 课程空间（论坛）仍为共享，同一课程的所有用户均可参与讨论
- 移除前端 `handleEnroll` 和课程卡片中的"已加入/加入课程"状态显示

### 6. 课程去重规则调整
- 去重依据：`课程号（description 中） + 授课教师`
- 不设立学期字段限制，过往资料可持续复用
- 若课程已存在，仅将当前用户加入该课程（`user_courses`），不重复创建

### 7. 导入前使用须知
- 点击"导入课程表"按钮后，先弹出"使用须知"弹窗
- 读取 `pre-notes.md` 文件，以 markdown 格式渲染
- 底部两个按钮："不同意"（关闭弹窗）、"我已同意并知晓"（进入导入界面）
- 只有点击同意后才能进入文件选择和导入流程

### 8. Markdown 渲染修复
- `renderMarkdown()` 函数优先使用 `marked.parse()` 渲染
- 移除 try-catch 包裹（避免吞掉加载延迟导致的 fallback）
- 添加 `marked.js` 未加载时的 console.warn 提示
- `pre-notes.md` 和 `notes.md` 中的 `**加粗**` 等语法现可正确渲染

## 课程导入数据流

```
点击"导入课程表"
  → 弹出使用须知（pre-notes.md）
  → 用户点击"我已同意并知晓"
  → 弹出导入界面（notes.md + 文件选择）
  → 选择 xlsx 文件
  → POST /api/schedule/import [Auth]
  → 限流检查（100天 / 50门）
  → parseSchedule() 解析
  → 按 courseId+teacher 去重
  → 新课程 → INSERT INTO courses + user_courses
  → 已有课程 → INSERT INTO user_courses（加入）
  → 返回当前用户的课程列表
  → 前端 navigateTo('courses') 刷新
  → 课程卡片点击 → 课程空间论坛
```
