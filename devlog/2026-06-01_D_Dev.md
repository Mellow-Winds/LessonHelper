## 后端变更

### 数据库迁移（server.js）

新增 users 表列：
- `wechat` TEXT — 微信号
- `douyin` TEXT — 抖音号
- `avatar_desc` TEXT — 个人肖像描述（80字以内）
- `mbti` TEXT — MBTI 人格类型
- `checkin_streak` INTEGER — 连续签到天数
- `last_checkin_date` TEXT — 上次签到日期
- `grace_days` INTEGER — 缓冲剩余天数

新增表：
- `follows` — 关注关系（follower_id, following_id, UNIQUE 约束）
- `feedback` — 问题反馈（user_id, category, content, contact）

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/checkin` | 每日签到（含双重保护机制） |
| GET | `/api/user/:id/profile` | 公开名片（含关注计数 + 隐私过滤） |
| POST | `/api/user/:id/follow` | 关注用户 |
| DELETE | `/api/user/:id/follow` | 取消关注 |
| GET | `/api/user/:id/followers` | 粉丝列表 |
| GET | `/api/user/:id/following` | 关注列表 |
| POST | `/api/user/feedback` | 提交问题反馈 |

### 签到保护机制

- **短期保护**：连续 7 天以上，漏签 3 天内签到可重燃
- **长期保护**：连续 45 天以上，漏签 7 天内签到可重燃
- **归零重签**：缓冲期过则 streak 归 1，重新开始

## 前端变更

### 路由系统（router.js）

新增路由：
- `/profile` → `profile`（主页）
- `/profile/edit` → `profile-edit`（编辑资料）
- `/profile/privacy` → `profile-privacy`（隐私设置）
- `/profile/data` → `profile-data`（数据管理）
- `/profile/:id` → `profile-user`（查看他人主页）

### 三栖视角系统

1. **Owner 编辑视角**：自己看自己的主页，字段可原位编辑（点击文字直接修改）
2. **Owner 预览视角**：一键模拟他人视角，字段锁定为只读，底部浮动提示条
3. **公开主页视角**：查看他人名片，显示关注按钮，隐私锁定时显示锁屏卡片

### 核心组件

- **签到勋章**：火焰图标 + 连续天数 + 缓冲期倒计时
- **完整度进度条**：7 项因子（昵称/专业/年级/QQ/微信/肖像描述/MBTI）等额权重
- **MD3 设置列表**：左侧图标 + 标题 + 描述 + 右箭头，点击进入子页面
- **原位编辑**：点击字段值 → 内联输入框 → Enter 保存 / Escape 取消
- **MBTI 选择器**：4×4 网格下拉，点击即选即保存
- **关注系统**：关注/取关按钮 + 粉丝/关注计数 + 点击查看列表
- **问题反馈**：模态框，类型选择 + 内容输入 + 联系方式

### 样式系统（style.css）

新增组件样式：
- `.profile-page` / `.profile-card` — 仪表盘布局
- `.profile-avatar` — 头像组件
- `.profile-streak-badge` — 签到勋章
- `.profile-completion-*` — 完整度进度条
- `.profile-field-row` — 资料字段行（含原位编辑态）
- `.profile-stats-row` — 关注/粉丝统计
- `.profile-list-item` — MD3 设置列表项
- `.profile-toggle-row` — 隐私开关行
- `.profile-data-grid` — 数据概览网格
- `.profile-follow-*` — 关注列表组件
- `.profile-preview-banner` — 预览模式浮动条
- `.profile-empty-*` — 空状态卡片
- `.profile-mbti-dropdown` — MBTI 选择器

## 设计原则

- 遵循 themerules：无渐变、无毛玻璃、无 emoji、纯实体色块
- 图标统一使用 Material Icons（`.mi`）+ Remix Icon（品牌图标）
- 所有输入框使用 `createMdInput()` / `createMdSelect()` 工厂
- 零全局命名空间污染，全部逻辑封装在 profile.js 模块沙箱内
- 面包屑返回机制：所有子页面标配 `arrow_back` + 面包屑路径
